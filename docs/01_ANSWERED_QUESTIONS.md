## Donation flow

- **OQ-13.** Monobank Banka — як підтверджуємо, що користувач задонатив саме від цього email? Варіанти:
  - (а) Користувач завантажує квитанцію → ми створюємо тікет адміну, ручна верифікація — низький автоматизм, прийнятно для MVP.
  - (б) Monobank Acquiring (платіжний шлюз з webhook) — повноцінна автоматизація, але потрібен мерчант-акаунт.
  - **Запропонований default:** на MVP — гібрид: показуємо банку (швидко й просто) + форма «я задонатив, ось скрін» + автоматичне підтвердження через Monobank Acquiring webhook, коли підключимо мерчант. --

## Інфраструктура

- **OQ-15.** Хостинг — **Mirohost Cloud, тариф MS21** (2 vCPU / 4 GB RAM / 40 GB SSD, ДЦ Київ «EU-IEV-1»). Продукт Cloud дає root-доступ → Docker ставимо самі. Наслідки: CAD-worker concurrency=1, потрібен swap, артефакти/бекапи → R2 (40 ГБ замало для локального зберігання), GDPR — дані в Україні («третя країна»). Деталі — ADR-011.
- **OQ-16.** **Так, заводимо акаунт Cloudflare.** Після переходу на Mirohost R2 для артефактів і бекапів став обов'язковим, не опційним. CF також лишається для DNS + проксі + SSL + WAF (геоблок RU/BY). Деталі — ADR-008, ADR-011.
- **OQ-17.** Discord-сервер для community — створено? Лінк у `README.md` як placeholder.
