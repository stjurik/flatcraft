ЕТАП 3 — конвеєр нової Деталі (гілка feat/new-part-pipeline)
A. docs/18_NEW_PART_SPEC.md — шаблон специфікації Деталі (форма для yurii):

1.  ідентифікація (назва UA/EN, slug, тип: деталь/preset);
2.  геометрія (креслення/ескіз/STEP; сегменти; гиби: кількість/кути/
    напрям/радіуси);
3.  таблиця параметрів (назва/од./min/max/default/крок; editable vs fixed;
    групи ADR-017);
4.  отвори/перфорація; 5) специфічні обмеження і cross-field правила;
5.  матеріали/товщини; 7) продуктовий шар (fixed params, use cases, фото);
6.  КОНТРОЛЬНИЙ ПРИКЛАД: набір параметрів + інженерний розрахунок
    розгортки (габарити, довжини сегментів) — стане обов'язковим
    unit-тестом нового шаблону (звірка з точністю 0.1 мм).
    Кожен пункт: що обов'язкове, що опційне, приклад заповнення
    (l_bracket як зразок).
    B. docs/promts/new-part-run.md — переюзабельний промпт додавання Деталі:
    вхід = заповнений 18-spec; кроки = TS definition + Python модуль +
    снапшоти + seed + контрольний приклад як тест + conformance + e2e +
    preview-PNG; вихід = 1 draft PR; STOP-правила ті самі; «Як перевірити
    очима» = відкрити студію нової деталі, покрутити параметри, експорт,
    звірити контрольний приклад.
    C. docs/12 §4 (DoD) — звірити з реальністю після міграції, розбіжності
    виправити; посилання на 18-spec і new-part-run.
    CI зелений → merge.
