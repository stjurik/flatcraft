### Патерн реалізації i18n (Privacy & Terms)

1. **Структура роутингу**: Використовується окрема підпапка `/en` всередині існуючих роутів (наприклад, `app/privacy/page.tsx` для UA та `app/privacy/en/page.tsx` для EN). Це створює окремий route для англійської версії.
2. **Спільні компоненти/дані**: Повне дублювання JSX та масивів з даними. Спільних компонентів або словників між UA та EN версіями немає. Наприклад, масив `SECTIONS` повністю продубльований і локалізований прямо в кожному файлі. Спеціальний i18n-фреймворк не використовується (відповідно до коментарів, щоб уникнути зайвих залежностей).
3. **Перемикання мови**: Реалізовано через прямі посилання (`<Link href="...">`), вбудовані в описовий текст під головним заголовком сторінки (в межах `<header>`). Наприклад: `English: <Link href="/privacy/en" ...> /privacy/en </Link>`.
4. **Metadata та hreflang**: Метадані (`title`, `description`) захардкоджені окремо в кожному файлі через експорт `metadata`. Спеціальних тегів `hreflang` або `alternates` для SEO-зв'язування мовних версій наразі немає.
