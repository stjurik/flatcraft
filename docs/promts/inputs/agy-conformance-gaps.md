# Прогалини у conformance-suite (Етап 1)

Порівняно з вимогами (docs/12 §3 та CLAUDE.md §7), наступні інваріанти наразі **НЕ покриваються** тестами в існуючій conformance-suite (`packages/templates/test/`):

## docs/12_TEMPLATE_CONTRACT.md §3

1. **Schema parity TS↔Python (property-based):** Повноцінні крос-мовні тести (з генерацією `fast-check` у TS та перевіркою `Pydantic` у Python) відсутні. Наразі є лише базова JSON round-trip перевірка дефолтних параметрів проти власної Zod-схеми.
2. **DXF/PDF детермінізм:** Відсутні перевірки на те, що фіксований seed гарантує сталу послідовність байтів на виході (DXF/PDF).
3. **e2e smoke (Playwright):** Повністю відсутні e2e тести для рендерингу ендпоінтів `/templates/{slug}`.
4. **React-free import у apps/api:** Відсутня статична чи динамічна перевірка, що імпорт шаблонів до `apps/api` не тягне за собою React-залежності.

## CLAUDE.md §7 (CAD-обмеження)

Існуюча suite (`registry-invariants.test.ts`) перевіряє render-gate лише для "явно неможливої товщини" (напр., 999). Жодна з реальних бізнес-перевірок на межах не покрита:

1. **Товщина у дозволеному діапазоні:** Не перевіряється, чи правильно валідуються реальні граничні значення товщини з `capability_matrix` для заданого матеріалу.
2. **Мінімальний радіус гиба:** Відсутні перевірки того, що `bend_radius_mm` ≥ `allowed_inner_radius_mm` для конкретної комбінації "матеріал + товщина".
3. **Мінімальна полиця після гиба:** Не перевіряється, що кінцева довжина полиці (напр. `legA_mm` або `rib_height_mm` після згину) ≥ `min_flange_mm` (7.5 мм).
4. **Габарити заготовки:** Немає тестів на те, що габарит не перевищує 3050 × `max_bend_length_mm` (з урахуванням товщини).
5. **Напрям згину (UP/DOWN):** Відсутня перевірка обов'язкового задання напрямку для кожного згину.
