# Рекомендації для Claude Code щодо виклику Antigravity (agy)

Цей документ містить інструкції для Claude Code (в ролі Оркестратора/Архітектора/Будівельника) щодо того, як ефективно делегувати завдання локальному агенту Antigravity (Gemini CLI, команда `agy`).

## 1. Хто такий Antigravity (`agy`) і навіщо його викликати

`agy` — це потужний локальний AI-агент (на базі Gemini 3.1 Pro). Ти (Claude Code) можеш взаємодіяти з ним через термінал (Bash).

Переваги `agy`, які доповнюють твої можливості:

- **Роль "Рев'юер" (ADR-036 §3)**: Забезпечує незалежне рев'ю коду (оскільки `agy` — це інша модель, що працює в окремому процесі).
- **Генерація UI (Зображень)**: `agy` має інструмент `generate_image`, який дозволяє створювати візуальні мокапи інтерфейсів (чого ти не можеш робити).
- **Актуальний веб-пошук**: `agy` вміє шукати інформацію в Інтернеті через `search_web` та читати сторінки `read_url_content`.
- **Фонові під-агенти (Subagents)**: `agy` нативно підтримує запуск паралельних під-агентів і таймерів/cron-задач у фоні.
- **Точковий масштабний рефакторинг**: `agy` має спеціалізований інструмент `multi_replace_file_content` для одночасної заміни багатьох фрагментів в одному файлі.

## 2. Коли делегувати завдання до `agy`

1. **AI-Рев'ю PR'ів (Level: Reviewer)**:
   Коли ти завершив TDD цикл і підготував draft PR, ти можеш викликати `agy` для перевірки коду на відповідність інваріантам з `CLAUDE.md §7` та загальним архітектурним правилам.
2. **Дизайн та Візуалізація**:
   Перед версткою нових сторінок (наприклад, мобільних форм), попроси `agy` згенерувати мокап-зображення для погодження з `yurii`.
3. **Глибока інвентаризація (Level: Scout/C-промпти)**:
   Якщо потрібно проаналізувати 50+ файлів на наявність патернів (наприклад, аудит логів або відмінностей шаблонів CAD), виклич `agy` у фоні, щоб не забивати свій власний контекст.

## 3. Як правильно викликати `agy` (Синтаксис Bash)

Оскільки ти (Claude) працюєш у headless-режимі, **ніколи не викликай `agy` в інтерактивному режимі**. Завжди використовуй прапорець `-p` (`--print`) або `--mode accept-edits`.

**Паттерн 1. Незалежне Code Review (через stdin пайп)**

```bash
git diff origin/main...HEAD | agy -p "Ти — незалежний Рев'юер. Прочитай diff та перевір його на відповідність інваріантам (CLAUDE.md §7). Збережи вердикт у docs/promts/inputs/review-result.md."
```

**Паттерн 2. Файлова передача контексту (Ресерч / Розвідка)**

```bash
agy -p "Проскануй файли apps/web/src/components/ та склади таблицю відмінностей у docs/promts/inputs/ui-audit.md. Використовуй свої інструменти для читання директорій."
```

**Паттерн 3. Виклик для генерації мокапу**

```bash
agy -p "Згенеруй зображення-мокап мобільної форми (/f/[exportId]) за дизайн-токенами проєкту. Збережи файл як артефакт і опиши його текстом у docs/promts/inputs/mockup-ready.md."
```

**Паттерн 4. Автономний запуск із дозволом на редагування**
Якщо `agy` має змінити код самостійно (наприклад, виконати складний `multi_replace_file_content`), використовуй безпечний пропуск дозволів для інструментів (лише якщо це ізольована гілка):

```bash
agy --dangerously-skip-permissions -p "Онови усі імпорти react-three-fiber у пакеті ui відповідно до нового стандарту."
```

**Вибір моделі (Model Selection)**
За замовчуванням `agy` використовує модель, вказану в глобальних налаштуваннях (`~/.gemini/antigravity-cli/settings.json`, ключ `"model"`).
Ти можеш явно перевизначити модель для конкретного виклику за допомогою прапорця `--model`:

```bash
agy --model "Gemini 3.1 Flash" -p "Зроби швидке рев'ю цього файлу"
```

_(Щоб побачити список усіх доступних моделей, виконай `agy models`)_

## 4. Обмеження та правила безпеки

- **Гонка за файли**: Якщо ти передаєш завдання `agy`, переконайся, що він зберігає результат у нейтральній зоні (наприклад, `docs/promts/inputs/`), або дочекайся завершення його роботи (`wait`), щоб ви не редагували один і той самий файл одночасно.
- **Уникай `--dangerously-skip-permissions` на головних гілках**: Використовуй цей прапорець лише в git worktree або в окремих feature-гілках, щоб `agy` не зробив неочікуваних змін без контролю.
- **Таймаути**: Враховуй, що за замовчуванням `agy -p` має таймаут (наприклад, 5 хвилин). Якщо завдання важке, май на увазі, що процес може перерватися.
