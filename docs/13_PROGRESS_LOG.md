## Feature 2.9.e — ізометрія виробу у PDF (2026-06-16)

- **Завершено** (2026-06-16, ADR-025): додано **довідкову векторну ізометрію** згорнутого виробу у праву колонку PDF-креслення під таблицею гибів/BOM (над QR). Стиль — інженерний **каркас**: видимі ребра суцільні (чорні 0.4pt), приховані пунктирні (сірі 0.3pt, dash 2,2), **з отворами** (узгоджено з користувачем). **Підхід (TDD):** не малюємо проєкцію вручну (HLR для згорнутої геометрії + отворів крихкий), а використовуємо вбудований OpenCascade hidden-line-removal, яким cadquery уже володіє. Worker будує 3D-solid через `build_<tpl>(params)` для кожного шаблону, але раніше **відкидав** його (`_ = build_<tpl>(...)`) — тепер прокидаємо solid у `export_<tpl>_pdf(..., solid=...)`. **(1)** Новий `export/isometric.py`: `project_isometric(shape, direction=(1,1,1))` повторює pipeline `getSVG` (`HLRBRep_Algo` → `HLRAlgo_Projector` → `Update/Hide` → `HLRBRep_HLRToShape`), розділяє `VCompound`/`OutLineVCompound` (visible) і `HCompound`/`OutLineHCompound` (hidden), дискретизує ребра (`GCPnts_QuasiUniformDeflection`, fixed tol 1e-3) у 2D-полілайни; `fit_to_box` вписує зі збереженням пропорцій. **(2)** Виявлено: `build_<tpl>` **навмисно не вирізає отвори** у 3D (preview = чистий extrude; отвори лише у DXF/розгортці) → новий `export/isometry_solid.py` згортає `unfolded.holes` (єдине джерело істини, координати розгортки) назад на 3D-грані й вирізає циліндри одним boolean-викликом (`Shape.cut`) — ефективно навіть для сітки перфо-панелі. Пер-шаблонний мапінг: `corner_angle` (сегмент B ⟂ Z, A ⟂ X; −Y через extrude вздовж −Y), `wall_shelf` (mount holes на back ⟂ X), `perforated_panel` (плоска ⟂ Y). **(3)** `export/pdf.py`: `_draw_isometric` (slot `(178, 52)` мм, бокс `96×44`, заголовок «Ізометрія (довідково)») + новий kwarg `solid` у всіх 5 `export_<tpl>_pdf`; оновлено module-docstring (ізометрія більше не «відкладена»). **(4)** `server.py`: 5× `_ = build_<tpl>` → `solid = build_<tpl>` з пробросом. **Детермінізм** збережено (HLR + дискретизація детерміновані, геометрія — чиста функція params) — byte-determinism тест зелений з ізометрією. **Візуально перевірено** (matplotlib-рендер полілайнів): `corner_angle`/`wall_shelf`/`perforated_panel` показують отвори на коректних гранях, приховані ребра пунктиром. **Тести:** 235 → **241 pytest** (+6: `test_isometric.py` — visible/hidden непорожні, детермінізм, отвори додають полілайни, `fit_to_box` bounds; +оновлено `test_pdf_export` helpers під `solid=`); mypy strict + ruff чисто (scoped override `OCP.*` без stubs). Гілка `feat/pdf-isometric-view`.

## UX-твік: уніфіковане групування параметрів шаблонів (2026-06-09)

- **Завершено + задеплоєно на staging** (2026-06-09, PR #21): уніфікація секцій редактора параметрів у всіх 5 студіях до єдиного набору категорій у фіксованому порядку **Матеріал і товщина → Розміри → Гиб → Сітка отворів**. Раніше кожен шаблон мав власні групи (`Полиця A`, `Полиця B`, `Верхня/Нижня/Середня полиця`, `Задня полиця`, `Фронт-губка`, `Лист`, `Гиби`, `Загальне`, `Отвори монтажу`) — UI виглядав неконсистентно між шаблонами. **Механізм не змінювався** — правлено лише значення `group:` у Zod `.describe("group:G|label:L")` (AutoForm групує за цими метаданими, ADR-017); порядок груп визначається першою появою поля в схемі, тож після перейменування скрізь виходить потрібна послідовність без змін коду AutoForm. **Зміни по групах:** усі розміри готового виробу → одна група «Розміри»; «Гиби» → «Гиб» (узгоджено й `bends.ts` `BendSpecSchema.direction`); «Отвори монтажу» (wall_shelf) → «Сітка отворів». **L/Z-кронштейни лишають «Отвори»** — отвори там вручну розставлені (масив координат), а не сітка, тож «Сітка отворів» було б технічно невірно. Кілька лейблів уточнено після злиття в одну «Розміри» групу (z_bracket: `Довжина (мм)`×2 → `Довжина верхньої/нижньої полиці`, `Вертикальний offset` → `Вертикальний offset середньої полиці`; wall_shelf: `Висота back-стінки` → `Висота задньої стінки`, `Висота губки` → `Висота фронт-губки`). **Не зачіпає** CAD-вивід (DXF/PDF), серверну/клієнтську валідацію (ADR-019/022), `data-testid` полів (testid'и походять від name поля, не від групи/лейбла) — суто косметичне перегрупування форм. **Файли:** 6 схем `packages/types/src/templates/*` (corner-angle/l-bracket/z-bracket/wall-shelf/perforated-panel/bends) + e2e `apps/web/tests/e2e/material-and-grouping.spec.ts` (легенди corner_angle 5→4 групи). **Тести зелені:** types 73, web unit 33, повний CI (typecheck/lint/Playwright e2e/python/build). Деплой стандартним флоу `release.yml`→`deploy-staging.yml` (pinned `sha-db05a4d`), smoke зелений; верифіковано наживо — `/templates/corner_angle` рендерить рівно 4 fieldset-групи у порядку Матеріал → Розміри → Гиб → Сітка отворів.
